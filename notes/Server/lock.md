# 多线程使用锁经验总结

## 减少锁的使用次数

使用锁带来性能损失：

- 加锁解锁本身有一定的开销

- 临界区代码不能并发

- 获取锁失败后会阻塞并让出 CPU，导致上下文切换的次数远多于不使用锁的次数

替代锁的方式很多，比如无锁队列

## 减少锁的使用粒度

减小锁的使用力度，即减小锁作用的临界区代码范围，临界区代码范围越小，多个线程排队进入临界区的时间就会越短

## 避免死锁

- 在一个函数中如果有加锁，一定要在函数退出时解锁，并且每个退出路径都要解锁

- 在线程退出时一定要及时释放持有的锁

- 多线程请求锁的方向要一致，避免死锁

- 当需要同一个线程重复请求一个锁时，需要明白使用锁的行为是递增锁引用计数，还是阻塞还是获得锁
