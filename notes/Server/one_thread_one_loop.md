# one thread one loop

线程指网线线程，即在网络线程函数里面都有一个循环，每个循环都做同样的事

## 循环部分

1. 步骤一：利用 I/O 复用技术分离出 socket 的读写时间

2. 步骤二：处理读写事件；对于监听 socket，读事件一般就是接收新连接；对于普通 socket，读事件就是调用 `recv` 或 `read` 接收数据；socket 写事件一般是发送数据

3. 步骤三：解包等

## 线程分工

网络线程一般分为网络主线程和工作线程

网络主线程只需处理新的连接，不用处理网络 I/O，新的连接根据一定的负载均衡策略分配给工作线程

网络主线程调用 `accept` 产生新连接客户端的 socket，并将 socket 传递给工作线程，由工作线程处理网络 I/O 事件

工作线程循环：

1. 通过 `select`、`poll` 或 `epoll` 检测网络时间

2. 检测到事件后，进行数据的收发

3. 执行钩子函数

## 超时时间设置

如果将 `poll`、`select` 或 `epoll` 的超时时间设置为 0，那么在没有网络 I/O 事件时工作线程将会空转浪费 CPU 时间片

如果将 `poll`、`select` 或 `epoll` 的超时时间设置为大于 0，那么在没有网络 I/O 事件却有其他事件要处理时，不能及时处理其他时间

正确做法：给 `select`、`poll` 或 `epoll` 设置一个超时时间，但对于其他事件，采取一种特殊的唤醒策略

可以在 `select`、`poll` 或 `epoll` 中挂载一个 fd，当需要唤醒时向这个 fd 写入数据

## 效率保障

在整个网络 I/O 线程中，除了 I/O 复用函数可能会造成等待，其他流程中不能有阻塞整个线程或耗时的操作

如果业务决定了有耗时操作，应该新开业务线程去处理耗时操作，I/O 线程本身不应该处理耗时操作
