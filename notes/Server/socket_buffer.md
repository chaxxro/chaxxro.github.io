# 缓冲区

## 必要性

- 由于 TCP 窗口太小，会导致数据无法发送出去，而上层可能不断产生新数据，从而需要将数据先存储起来，因此发送数据时需要发送缓冲区

- 接收到数据时：

1. 除了一些通用协议，大部分业务使用自定义协议格式，因此对数据进行解包应该由业务层完成，网络通信层一般不处理数据解包

2. 由于 TCP 是流式协议，某一次收到的数据长度不一定够完整的包长，因此需要接受缓冲区先将数据存储起来

3. 一些特殊业务要求先将数据存储起来，等满足一定条件再取出处理

因此网络层需要一个接收缓冲区和发送缓冲区

## 设计

缓冲区一般是一个内存连续的容器，一般需要两类接口：存储数据接口和获取数据接口

对于发送缓冲区，由于上层交给网络层的数据是有序的，若某次需要发送的数据未发完，剩余的数据一定排在后续产生的数据前面

对于接收缓冲区，由于不断从 socket 读取数据，后面读到的数据一定排在前面读到数据的后面

缓冲区的容量可以是动态增加的，按需分配

缓冲区的写入和读取接口需要考虑不同数据类型，如 `char`、`short` 等

对于接受缓冲区，需要提供寻找特殊标志的接口，如提供 `\n` 的接口

缓冲区提供同时读、写的功能，因此需要读写两个指针来标明位置

## 注意

对于服务端来说，由于需要同时服务多个客户端，不同连接的 socket 会有自己的接收缓冲区和发送缓冲区

由于服务端需要支持多个连接，为了节省内存，所以缓冲区的容量一般是有上限的

对于发送缓冲区，如果较长时间未发送数据，可以认为连接断开，需要将缓冲区回收
