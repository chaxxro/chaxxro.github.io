# 定时器设计

最简单的定时器设计是直接使用 `sleep` 函数

## 基本思路

定时器对象需要一个唯一标识、过期时间、重复次数、定时器触发动作

定时器回调函数不能有耗时或者阻塞线程的操作

唯一标识需要全局唯一，可以设置为 Timer 对象的静态变量，利用原子操作获取唯一标识

## 性能优化

### 定时器对象集合的数据结构优化

最简单的定时器容器是使用 `list`，每次检测定时器对象是否触发都需要遍历整个 `list`

1. 在将定时器添加至集合时，可以对集合进行排序，这样在遍历定时器集合时可以提前停止遍历，但移除定时器仍需要遍历全部 `list`

2. 使用时间轮，将现在时刻 t 加上一个时间间隔 interval，以 interval 为步长，将各个定时器对象的过期时间按步长分布在不同的时间槽中，时间槽中按加入槽的顺序串成 `list`；因为时间槽间隔是固定的，在每次检测时判断当前时间处于哪个时间槽，比槽序号小的全部到期

3. 使用时间堆，用小顶堆来管理定时器对象，按过期时间从小打到排序

### 对时间的缓存

获取系统时间是系统调用，为提高效率，在一些对时间要求精度不高的情况下可以缓存时间
